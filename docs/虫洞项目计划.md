# 虫洞文件传输 —— 项目计划与进度

> 最后更新：2026-06-13
> 一句话目标：把文件拖进桌面上的「虫洞」桌宠，文件自动出现在另一台电脑上。

## 总体设计（已定稿）

需求选择（已和自己确认）：

- 网络环境：**同一局域网**（两台电脑连同一 WiFi/路由器，其中一台或第三台当 FTP 服务器）
- 传输方向：**双向同步**（任意一台拖入，另一台都收到）
- 实现方式：**基于现有 FTP 服务器**（复用已有成果）
- 文件落点：**固定的「虫洞收件箱」文件夹**（默认 `~/Wormhole/收件箱`，可配置）
- 桌宠形态：**黑洞吞噬感 + 乳白色吸积盘 + 桌面小图标大小（低调）**

架构（两层解耦）：

```
   电脑A                      FTP服务器                      电脑B
┌──────────┐            ┌────────────────┐            ┌──────────┐
│ 虫洞桌宠   │  ①拖入文件  │   /wormhole     │  ②轮询发现   │ 虫洞桌宠   │
│ (吸入动画) ─┼─上传──────>│   共享频道中转   │──下载──────>│ (喷出动画) │
│ 收件箱 <───┼─下载────────│ (FTP STOR/RETR) │<──────上传──┼── 拖入文件 │
└──────────┘            └────────────────┘            └──────────┘
   面子:桌宠(QML动画+拖拽)      里子:FTP服务器(已做好)       面子:桌宠
```

## 快速命令备忘

```bash
cd <项目目录>

# 测试同步引擎（无需 GUI）
PYTHONPATH=src python3 tests/test_wormhole.py

# 起 FTP 服务器
PYTHONPATH=src python3 -m pyftp_server --host 0.0.0.0 --port 2121

# 起桌宠（需 PySide6）
PYTHONPATH=src python3 -m pyftp_server.wormhole.pet --host 127.0.0.1

# 命令行版同步（无 GUI，监视发件箱）
PYTHONPATH=src python3 -m pyftp_server.wormhole.sync \
    --host 127.0.0.1 --inbox ~/Wormhole/收件箱 --outbox ~/Wormhole/发件箱

# 推送
git push
```
