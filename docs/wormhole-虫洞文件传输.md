# 虫洞文件传输（Wormhole）

> 在 FTP 服务器之上做的一个小应用：把文件拖进桌面上的「虫洞」桌宠，文件就会自动出现在另一台电脑上。

## 这是什么

一个"无障碍"文件互传通道。两台电脑各放一个虫洞桌宠（黑洞造型、乳白色吸积盘、桌面小图标大小），都连到同一台 FTP 服务器的共享频道 `/wormhole`。你在任意一台把文件拖进虫洞，它先被上传到服务器中转，另一台几秒内自动下载到收件箱并播放"喷出"动画。

体验：**拖进去 → 黑洞收缩吸入 → 另一台黑洞膨胀吐出 → 文件落在收件箱**。

## 原理与架构

虫洞分为两层：底层复用课程设计的 FTP 服务器当中转站（里子），上层是同步引擎 + 桌宠挂件（面子）。

```
   电脑A                      FTP服务器                      电脑B
┌──────────┐            ┌────────────────┐            ┌──────────┐
│ 虫洞桌宠   │  ①拖入文件  │   /wormhole     │  ②轮询发现   │ 虫洞桌宠   │
│ (吸入动画) ─┼─上传──────>│   共享频道中转   │──下载──────>│ (喷出动画) │
│           │            │ (FTP STOR/RETR) │            │ 收件箱出现 │
│ 收件箱 <───┼─下载────────│                │<──────上传──┼── 拖入文件 │
└──────────┘   ④轮询发现   └────────────────┘   ③上传      └──────────┘
```

- **上传**：用 FTP `STOR` 把文件传到频道，先传 `名字.part` 再 `RNFR/RNTO` 改成正名，避免对端看到半截文件。
- **下载**：每隔约 2 秒列出频道（`LIST`/`SIZE`），凡是"已收清单"里没有的新文件就 `RETR` 下载到收件箱（同样先写 `.part` 再改名）。
- **去重 / 防死循环**：每个文件以「文件名|大小」为指纹存入 `.wormhole_seen.json`。自己发出的、已下载的都登记进去，轮询时只处理没见过的指纹——所以下载下来的文件不会被再传回去。
- **阅后即焚**：接收方下载成功后顺手 `DELE` 删除服务器上的中转副本，**中转站不留存文件**，隐私友好。文件在服务器上只存在「对方还没收到」的窗口期。两台以上设备同时入会时用 `--no-burn` 关闭（否则第一个下载的设备会把文件"焚"掉，其他设备收不到）。
- **断线重连**：连接异常自动重连，不中断同步。

模块位置：

```
src/pyftp_server/wormhole/
├── sync.py        同步引擎(上传/轮询下载/去重/重连)——纯后台, 已自动化测试
├── pet.py         桌宠挂件(PySide6+QML 动画 + 拖拽)——需图形界面
├── wormhole.qml   黑洞虫洞的视觉与动画
└── __init__.py
tests/test_wormhole.py   同步引擎端到端测试(双向/去重/无回环/回调)
```

## 准备：两台电脑 + 一台服务器

最简单的局域网用法：两台电脑连同一个 WiFi/路由器，其中一台（或第三台）当 FTP 服务器。

服务器上虫洞共享频道的账号（已内置）：

```text
用户名: wormhole
密码:   wormhole
频道目录: <FTP总根>/wormhole
```

## 启动步骤（局域网）

**第 1 步：在服务器电脑上启动 FTP 服务器**

```bash
PYTHONPATH=src python3 -m pyftp_server --host 0.0.0.0 --port 2121
```

查出这台电脑的局域网 IP（两台客户机要用它连接）：

```bash
# macOS
ipconfig getifaddr en0
# Linux
hostname -I
```

假设查到是 `192.168.1.10`。

**第 2 步：在装了 PySide6 的电脑上启动虫洞桌宠**

先装依赖（只需一次；pyobjc 仅 macOS 需要，用于挂件常驻所有桌面）：

```bash
pip install PySide6 pyobjc-framework-Cocoa --break-system-packages
```

启动桌宠（把 host 换成服务器的局域网 IP）：

```bash
PYTHONPATH=src python3 -m pyftp_server.wormhole.pet --host 192.168.1.10
```

两台电脑都这样启动后，桌面右下角会出现一个黑洞小挂件（中心纯黑、外圈乳白渐变，大小约为系统程序图标的 1.5 倍，随 Dock 图标尺寸自适应）。

**第 3 步：传文件**

把任意文件从桌面拖到挂件上 → 黑洞收缩吸入（压缩感）→ 另一台几秒后黑洞膨胀吐出，文件出现在收件箱。

- 拖动挂件可挪到桌面任意位置。
- 右键挂件可退出。

## 收件箱在哪 / 怎么改

默认收件箱：`~/Wormhole/收件箱/`。收到的文件都稳稳落在这里，不会丢。

改成别的目录（比如直接放桌面）：

```bash
PYTHONPATH=src python3 -m pyftp_server.wormhole.pet --host 192.168.1.10 --inbox ~/Desktop
```

## 没有图形界面？用命令行版

同步引擎本身不依赖 GUI。可以用命令行版：监视一个"发送文件夹"，放进去的文件自动发送；收到的文件落入收件箱。

```bash
PYTHONPATH=src python3 -m pyftp_server.wormhole.sync \
    --host 192.168.1.10 --inbox ~/Wormhole/收件箱 --outbox ~/Wormhole/发件箱
```

把文件丢进 `发件箱` 就会自动上传；对端的文件会自动出现在 `收件箱`。

## 启动参数

| 参数 | 说明 | 默认 |
|------|------|------|
| `--host` | FTP 服务器地址（局域网填服务器内网 IP） | `127.0.0.1` |
| `--port` | FTP 端口 | `2121` |
| `--user` / `--password` | 频道账号 | `wormhole` / `wormhole` |
| `--inbox` | 收件箱目录 | `~/Wormhole/收件箱`（pet）/ `received`（sync） |
| `--outbox` | 监视发送目录（仅命令行版 sync） | 无 |
| `--interval` | 轮询间隔（秒） | `2.0` |
| `--no-burn` | 关闭阅后即焚（>2 台设备时使用） | 默认开启阅后即焚 |
| `--tls` | FTPS 传输加密（服务器需 `--tls` 启动） | 关 |
| `--secret 口令` | 端到端文件加密（两台电脑口令必须一致） | 关 |
| `--size 像素` | 挂件边长（仅 pet）；0=随系统 Dock 图标尺寸自适应（约图标 1.5 倍观感） | `0` |

## 加密（双层可叠加）

两层加密各管一段，可单独开也可同时开：

| 层 | 开关 | 防什么 | 原理 |
|----|------|--------|------|
| **FTPS 传输加密** | 服务器 `--tls` + 客户端 `--tls` | 网络窃听（WiFi 抓包、运营商、中间人） | RFC 4217 显式 TLS：`AUTH TLS` 升级控制连接，`PROT P` 加密数据连接 |
| **端到端文件加密** | 两台客户机 `--secret '口令'` | 服务器管理员偷看 | AES-256-GCM，密钥由口令经 PBKDF2 派生；服务器上躺着的永远是密文 |

```bash
# 服务器：生成自签证书并以 TLS 模式启动
make cert
PYTHONPATH=src python3 -m pyftp_server --host 0.0.0.0 --pasv-ip <公网IP> --tls

# 两台客户机：传输加密 + 端到端加密一起开(需 pip install cryptography)
PYTHONPATH=src python3 -m pyftp_server.wormhole.pet \
    --host <公网IP> --password '频道密码' --tls --secret '两边一致的加密口令'
```

注意：

- 端到端加密需要 `pip install cryptography`（仅客户机，服务器不用）。
- 自签证书场景客户端不校验证书链（防窃听已足够；防中间人需正式 CA 证书，课程范围内说明即可）。
- 两边 `--secret` 不一致时文件按密文保存不丢失，提示「解密失败」。
- TLS 暂不支持 `select` 并发模型（非阻塞 TLS 握手超出课程范围），用默认 thread 即可。

## 常见问题

**支持什么格式？** 全部。底层是 FTP 二进制流，逐字节搬运，不关心内容。限制只有三条：不能拖文件夹（先压缩成 zip）、`.` 开头的隐藏文件不同步、同名文件被新版本覆盖。

**网络要求？** 客户端零要求：家庭 WiFi/校园网/手机热点都行，不需要公网 IP、不需要端口映射——所有连接都是客户端→服务器单方向发起，NAT 天然穿透。只有服务器需要公网 IP 和放行端口。两台电脑也**不要求同时在线**：文件在服务器频道等着，对方上线自动补收（store-and-forward）。

**会一直监视吗？** 挂件运行期间每 2 秒轮询一次服务器频道（几百字节流量），右键退出后一切停止，无后台残留。它只看频道目录，不扫描你电脑上的任何文件；只有亲手拖进黑洞的文件才会上传。

**服务器存文件吗？** 默认阅后即焚：对方下载成功即删除中转副本。服务器上只在「已发出、对方还没收到」的窗口期内短暂存有文件。

**收件箱能换地方吗？** `--inbox ~/Desktop` 随便指。注意去重账本 `.wormhole_seen.json` 跟着收件箱走，换目录时把它一起拷过去可避免重复下载频道存量文件（开了阅后即焚后频道基本无存量，影响很小）。

## 跨网络：用云服务器当中转站

两台电脑不在同一局域网时，可以把 FTP 服务器架在云服务器上（虫洞流量只走「客户端 → 服务器」方向，天然穿透 NAT）。

**云服务器上：**

```bash
# 1) 改掉默认弱密码(公网必做)，再启动服务器并宣告公网 IP
export PYFTP_WORMHOLE_PASSWORD='你的强密码'
PYTHONPATH=src python3 -m pyftp_server --host 0.0.0.0 --port 2121 \
    --pasv-ip <公网IP> --tls --wormhole-only
# --tls           FTPS 加密(先 make cert 生成证书)
# --wormhole-only 只保留 wormhole 账号，关闭 admin/admin 等内置弱密码账号(公网必开)
```

**云控制台安全组放行**：TCP `2121`（控制连接）+ TCP `50000-50100`（PASV 数据端口）。

**两台客户机上：**

```bash
PYTHONPATH=src python3 -m pyftp_server.wormhole.pet --host <公网IP> --password '你的强密码'
```

> ⚠️ FTP 是明文协议，公网部署仅建议课程演示用：务必改强密码，演示后及时关闭服务器或限制安全组来源 IP。长期使用应升级 FTPS（见 v2 计划）。

## 当前版本范围

v1 聚焦把"拖进去→另一边出现"做扎实，因此：

- 只同步**文件**，暂不同步子文件夹（文件夹可压缩成 zip 后传）。
- 不同步**删除**（一台删了文件，不会让另一台也删——更安全，防误删）。
- 同名文件会被新版本覆盖。
- 默认**阅后即焚**（中转站不留存），多于两台设备时需 `--no-burn`。

这些都可在后续版本扩展。
